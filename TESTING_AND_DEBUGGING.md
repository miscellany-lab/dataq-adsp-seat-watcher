# Testing and Debugging Notes

이 문서는 메인 `README.md`와 분리된 테스트/디버깅 기록입니다. 프로젝트가 완성형 범용 감시기라는 의미가 아니라, 실제 DataQ ADsP 고사장 팝업을 다루며 발견한 문제와 보완 내용을 남기기 위한 기록입니다.

## 현재 검증 수준

- 실제 잔여좌석 사례 1건에서 OCR 감지와 Telegram 알림을 확인했습니다.
- 전체 좌석이 0석인 상태에서 2회 스캔했고, 알림이 발생하지 않는 것을 확인했습니다.
- 다른 지역, 다른 행, 다른 잔여좌석 숫자, 다른 화면 배율/해상도에서는 추가 검증이 필요합니다.
- OCR 기반이므로 오탐과 미탐 가능성이 남아 있습니다.

## 정책 경계

- 자동 접수, 자동 결제, 자동 로그인은 구현하지 않습니다.
- 캡차 우회, DataQ 자동화/개발자도구 감지 우회는 구현하지 않습니다.
- 비밀번호와 Telegram token은 저장하지 않습니다.
- 실제 접수와 결제는 사용자가 DataQ 사이트에서 직접 진행합니다.

## 대상 화면

- URL: `https://www.dataq.or.kr/www/components/map/list.do`
- 화면: DataQ `시험 고사장 목록` 팝업
- 표 컬럼: `No.`, `지역`, `고사장명`, `주소`, `엘리베이터 운영 여부`, `잔여좌석`, `지도`
- 감시 대상 시험: 제50회 데이터 분석 준전문가(ADsP)

## 주요 실행 명령

실사용 기준 명령:

```powershell
python adsp_popup_ocr_watcher.py --refresh --confirm-resubmit --interval 40 --pages 15 --scroll-method wheel --wheel-notches 9 --focus-click 900,500 --keep-awake --bbox 0,90,1845,980 --seat-bbox 1535,90,1645,980 --save-text ocr_debug.txt
```

0석 오탐 검증 명령:

## 디버깅 타임라인

### Playwright 방식 중단

초기에는 Python + Playwright로 DataQ 접수 화면의 `select`/`option`을 읽는 방식을 시도했습니다.

- `launch_persistent_context`
- Chrome 프로필 `./dataq_chrome_profile`
- 대상 URL: `https://www.dataq.or.kr/www/accept/list.do`

DataQ가 Playwright 또는 개발자도구 환경을 감지해 차단했기 때문에 이 방식은 중단했습니다. 감지 우회 코드는 작성하지 않기로 했습니다.

### 수동 클립보드 방식

사용자가 일반 Chrome에서 표 또는 option 텍스트를 직접 복사하고, 프로그램이 클립보드 텍스트에서 잔여좌석을 파싱하는 방식입니다.

- 장점: 브라우저 조작 없음
- 단점: 사용자가 직접 복사해야 함
- 파일: `adsp_manual_clipboard_watcher.py`

### 일반 화면 OCR 방식

일반 Chrome 화면을 사용자가 직접 열어둔 상태에서 OCR로 읽는 방식입니다.

- 장점: 표 복사가 필요 없음
- 단점: 화면이 갱신되지 않으면 사용자가 직접 새로고침해야 함
- 파일: `adsp_ocr_watcher.py`

### 팝업 OCR 방식

DataQ 고사장 목록 팝업을 전면에 둔 상태에서 화면 OCR, 스크롤, 선택적 새로고침, Telegram 알림을 조합했습니다.

- 파일: `adsp_popup_ocr_watcher.py`
- 기본 주기: `--interval 40`
- 스캔 화면 수: `--pages 15`
- OCR 영역: `--bbox 0,90,1845,980`
- 잔여좌석 열 보조 OCR: `--seat-bbox 1535,90,1645,980`

## 발견한 문제와 조치

### 새로고침 후 양식 다시 제출 확인

Chrome에서 팝업 새로고침 시 `양식 다시 제출 확인` 대화상자가 뜹니다.

조치:

- `--refresh`로 활성 창 새로고침
- `--confirm-resubmit`로 확인창을 Enter 처리
- `--confirm-retries`, `--confirm-wait`로 재시도 조정 가능

### 새로고침 후 스크롤 포커스 손실

새로고침 후 팝업 본문을 다시 클릭해야 휠 스크롤이 동작하는 문제가 있었습니다.

조치:

- `--focus-click 900,500` 옵션 추가
- 새로고침 후 본문 좌표 클릭으로 포커스 복구

### 스크롤이 끝까지 내려가지 않음

초기 스크롤 설정은 고사장 목록 전체를 충분히 훑지 못했습니다. 70번 근처까지 도달해야 하는 상황이 있었습니다.

조치:

- `--pages 15`
- `--scroll-method wheel`
- `--wheel-notches 9`
- 전체 스캔 후 다시 상단으로 복귀하는 흐름 조정

### 동작 간 딜레이가 너무 김

초기 딜레이가 길어 한 사이클이 느렸습니다.

조치:

- 확인 주기를 `60초`에서 `40초` 기준으로 조정
- 새로고침, 확인창 처리, 포커스 클릭, 스크롤 간 대기 시간을 줄임

### MessageBox가 브라우저 포커스를 뺏음

Windows `MessageBoxW` 알림이 Chrome 포커스를 뺏어 이후 스크롤 감시에 영향을 줄 수 있었습니다.

조치:

- 메시지박스는 기본 비활성화
- 필요할 때만 `--message-box`로 활성화
- 외출 중 감시에서는 Telegram 알림 중심으로 사용

### Telegram 알림 중복 발송

전처리나 스캔 중 같은 조건을 여러 번 읽으면 Telegram 알림이 반복 발송되는 문제가 있었습니다.

조치:

- 한 스캔 사이클에서 최초 감지만 알림
- `cycle_alert_sent` 방식으로 중복 발송 억제

### OCR이 제주도 테스트 키워드를 놓침

70번 행 `제주도` 키워드가 보이는 상황에서도 OCR 텍스트에서 안정적으로 잡히지 않았습니다.

확인:

- 화면상에는 제주도 행이 있었음
- OCR은 한글 지역명을 깨뜨리거나 누락할 수 있음

조치:

- 지역명 키워드만 믿지 않음
- 행번호와 `ADsP` 포함 여부, 마지막 숫자 후보를 함께 확인하는 보조 파싱을 추가

### 특정 잔여석을 OCR이 놓침

화면상 27번 행에 잔여좌석 `1`이 있었지만 OCR 원문에서 누락되는 사례가 있었습니다. 반면 64번 행 잔여좌석 `67`은 OCR 텍스트에서 다음처럼 깨진 줄로 잡혔습니다.

```text
64 SAaIoLA| ADsP (#4) SSSR AL OM Bae Nee 46 aso MESS 67
```

조치:

- `ADsP`가 포함된 OCR 행에서 마지막 정수 토큰을 좌석 후보로 확인
- 잔여좌석 열만 따로 OCR하는 `--seat-bbox` 보조 확인 추가
- 좌석 열 OCR은 단독 숫자만 인정하도록 필터를 강화

### 좌석 열 OCR 오탐 가능성

잔여좌석 열 보조 OCR은 행번호, 주소 숫자, 시간 표현 등을 잘못 좌석으로 인식할 수 있습니다.

조치:

- `--seat-bbox`를 잔여좌석 열로 좁힘
- digit whitelist 사용
- 공백 제거 후 `\d{1,3}` 형태의 단독 숫자만 인정
- `0`은 무시
- 문제가 있으면 `--disable-seat-column-check`로 비활성화 가능

## 양성 케이스 검증

### 실제 잔여석 감지 사례

실제 잔여좌석이 보이는 화면에서 다음 감지가 있었습니다.

```text
ADsP 잔여좌석 발견
OCR No.64 잔여좌석 67석
```

판정:

- 실제 화면에 잔여좌석 `67`이 존재했습니다.
- OCR 텍스트는 깨졌지만 행번호와 마지막 숫자 보조 파싱으로 감지했습니다.
- Telegram 알림 수신을 확인했습니다.

주의:

- 이 검증은 특정 화면 조건과 특정 행에서 성공한 사례입니다.
- 다른 지역, 다른 행, 다른 숫자 패턴에서도 같은 성능을 보장하지 않습니다.

## 음성 케이스 검증

### 전체 0석 상태 오탐 검증

증설 잔여석이 모두 소진된 상태에서 전체 팝업 목록을 2회 반복 스캔했습니다.

실행 로그:

```text
[2026-07-08 18:32:50] 활성 창 새로고침
[2026-07-08 18:32:50] 팝업 OCR 확인: 0건
[2026-07-08 18:34:30] 활성 창 새로고침
[2026-07-08 18:34:30] 팝업 OCR 확인: 0건
```

판정:

- 전체 잔여좌석 0석 상태에서 알림 없음
- Telegram 알림 없음
- 오탐 없음
- 2회 스캔 기준 통과

주의:

- 이 검증은 오탐 방지에 대한 확인입니다.
- 다른 조건에서 1석 이상이 발생했을 때 감지 성공을 보장하지는 않습니다.

## 현재 남은 검증 과제

- 다른 지역에서 1석 이상 발생 시 감지 여부
- 다른 행 위치에서 1석 이상 발생 시 감지 여부
- `1`, `2`, `10`, `67` 등 다양한 좌석 숫자 패턴 검증
- 화면 배율, 해상도, Chrome 창 크기 변화에 따른 안정성
- `--seat-bbox` 좌표가 다른 모니터/해상도에서도 맞는지 확인
- OCR 원문 누락 시 보조 좌석 열 OCR이 충분히 보완하는지 확인
- 오탐 발생 시 `ocr_debug_*.txt` 원문 기준으로 필터 추가

## 현재 결론

현재 프로젝트는 다음 수준으로 보는 것이 정확합니다.

```text
실제 잔여석 감지 및 Telegram 전송 여러 차례 확인 + 전체 0석 상태 오탐 없음 2회 확인.
다양한 양성 케이스가 추가로 확인되었지만, OCR 후보 수 과다와 좌석 수 흔들림은 계속 검증 필요.
```

따라서 이 도구는 취소표나 추가 증설 좌석 확인을 보조하는 실험적 로컬 도구이며, 접수 가능 여부나 감지 정확도를 보장하지 않습니다.
