# Testing and Debugging Notes

이 문서는 메인 `README.md`와 분리된 테스트/디버깅 기록입니다. 현재 제품 방향은 DataQ 자격검정 고사장 목록 전반을 대상으로 하지만, 실제 운영 검증은 ADsP 접수 기간의 DataQ 팝업을 기준으로 가장 많이 수행했습니다.

## 검증 범위

- 기준 화면: DataQ `시험 고사장 목록` 팝업
- 표 컬럼: `No.`, `지역`, `고사장명`, `주소`, `엘리베이터 운영 여부`, `잔여좌석`, `지도`
- 검증 사례: 제50회 데이터 분석 준전문가(ADsP) 접수 화면
- 목표: 동일한 DataQ 팝업 구조를 쓰는 다른 자격검정에서도 같은 파싱 흐름을 재사용

## 정책 경계

- 자동 접수, 자동 결제, 자동 로그인은 구현하지 않습니다.
- 캡차 우회와 DataQ 자동화/개발자도구 감지 우회는 구현하지 않습니다.
- Telegram token, chat_id, 비밀번호는 파일에 저장하지 않습니다.
- 사용자가 직접 연 Chrome 팝업의 화면과 클립보드 텍스트만 읽습니다.

## 핵심 변경 기록

### Playwright 방식 중단

초기에는 Python + Playwright로 DataQ 접수 화면의 select/option을 읽는 방식을 시도했습니다. DataQ가 Playwright/개발자도구 환경을 감지해 차단했기 때문에 이 방식은 중단했고, 감지 우회 코드는 작성하지 않기로 했습니다.

### OCR 단독 방식의 한계

OCR은 좌석 숫자 위치를 찾는 데 유용하지만 한글 고사장명과 주소를 자주 깨뜨렸습니다. 이 때문에 OCR 결과만 Telegram으로 보내면 `SAaIoLA`, `MESS` 같은 깨진 텍스트가 섞였습니다.

### 클립보드 보정 추가

현재 주 흐름은 OCR로 스크롤/좌석 후보를 감지하고, 활성 DataQ 팝업에서 자동 복사한 클립보드 텍스트로 행번호, 지역, 고사장명, 주소, 좌석 수를 정형화합니다. 이 방식으로 Telegram과 GUI 결과가 사람이 읽기 쉬운 형태가 되었습니다.

### 지역 필터 제거

초기에는 개인 사용 목적에 맞춰 특정 지역 우선순위와 제주 제외 조건이 있었지만, 배포 가능한 DataQ 범용 도구로 바꾸면서 개인 조건 필터는 제거했습니다. 필터링은 추후 사용자가 직접 선택하는 옵션으로 다시 설계할 예정입니다.

### 결과 정렬

GUI 결과는 고사장 목록의 `No.` 기준 오름차순으로 정렬합니다. 같은 주기에 여러 좌석이 잡혀도 원래 목록 순서대로 볼 수 있게 하기 위해서입니다.

## 주요 실행 명령

일반 사용자 GUI:

```powershell
python dataq_desktop_app.py
```

CLI 검증:

```powershell
python dataq_popup_ocr_watcher.py --refresh --confirm-resubmit --interval 40 --pages 15 --scroll-method wheel --wheel-notches 9 --focus-click 900,500 --keep-awake --bbox 0,90,1845,980 --seat-bbox 1535,90,1645,980 --save-text ocr_debug.txt
```

## 검증된 동작

- 0석 상태에서 반복 감시해도 알림이 발생하지 않음
- 잔여좌석이 있는 행만 감지
- Telegram 미설정 상태에서도 GUI 결과 카드에 표시
- Telegram 설정 시 같은 정형화 결과를 휴대폰으로 전송
- 클립보드 텍스트가 있으면 OCR 깨짐보다 클립보드 행 정보를 우선 사용
- ADsP 외의 DataQ 시험 코드 예시도 파서에서 처리 가능하도록 범용화

## 남은 검증 과제

- ADsP 외 DataQ 자격검정 팝업에서 실제 좌석 변동 검증
- 사용자가 원하는 지역/시험/좌석 수 조건 필터 UI 추가
- GUI 스타일 고도화
- 설치형 exe 빌드 흐름 정리
- 검증 영상과 스크린샷 추가
